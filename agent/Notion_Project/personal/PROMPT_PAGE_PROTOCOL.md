# AGENT_PROTOCOL: Prompt Engineering Page
> **Status**: Active | **Version**: v1.0 | **Role**: Protocol Officer
**적용 대상**: "프롬프트 엔지니어링" (Prompt Engineering) Notion Page
**목적**: AI 에이전트가 이 페이지와 하위 데이터베이스에 데이터를 입력할 때 **데이터 무결성**과 **안전성**을 보장하기 위함.

> [!IMPORTANT]
> **AI Agent 필독**: 이 페이지에 새로운 프롬프트를 등록하거나 수정할 때는 아래 프로토콜을 준수해야 합니다. 스키마에 맞지 않는 데이터 입력은 금지됩니다.

## 1. Schema Definition (데이터 구조)

이 페이지는 **프롬프트 버전 관리 시스템 (Prompt VCS)**으로 동작합니다. 아래 컬럼 스키마를 반드시 따르십시오.

| Property Name | Type | Allowed Values / Constraints | Default Value | Notes |
| :--- | :--- | :--- | :--- | :--- |
| **문서 이름** (Title) | Title | `[Topic] Description` 형식 권장 (예: `[Coding] Python Code Generator`) | (Required) | 고유한 식별자 역할을 해야 함. |
| **상태** (Status) | Select | `Draft` (초안), `In Progress` (작성중), `Stable` (사용가능), `Deprecated` (폐기) | `Draft` | 신규 생성 시 무조건 `Draft`로 시작. |
| **버전** (Version) | Text | SemVer 형식 (`v1.0`, `v1.1`, `v2.0-beta`) | `v0.1` | 단순 정수 입력 금지. 반드시 `v` 접두사 포함. |
| **Category** | Multi-Select | `Coding`, `Writing`, `Data Analysis`, `General`, `Creative` | (Required) | 관련 태그 최소 1개 선택 필수. |
| **Model Info** | Text | Model ID (예: `gpt-4-0613`, `claude-3-opus`) | `N/A` | 특정 모델에 최적화된 경우 기입. (Context7 Ref: Standard Metadata) |
| **최종 편집 일시** | Date | (Auto-generated by Notion) | - | 수정 시 자동 갱신됨. |

## 2. Validation Rules (입력 검증 규칙)

AI는 데이터를 입력하기 전에 다음 조건을 스스로 검증해야 합니다.

1.  **Version Consistency**:
    *   기존 문서의 내용을 수정할 경우, 기존 내용은 **보존**하고 **새로운 페이지**를 생성하여 `Version`을 올리십시오. (Immutable History)
    *   *예외*: `Status`가 `Draft`인 상태에서는 같은 페이지 내에서 수정 가능.
2.  **Status Workflow**:
    *   `Stable` 상태의 프롬프트는 함부로 수정하지 마십시오. 수정이 필요하면 복제 후 버전을 올리고 `Draft`로 시작하십시오.
    *   `Deprecated` 상태의 문서는 읽기 전용으로 취급하십시오.
3.  **Content Formatting**:
    *   **Prompt 본문**은 반드시 **Code Block** 또는 **Callout Block** 안에 작성하여, 설명 텍스트와 프롬프트 본문이 명확히 구분되도록 하십시오.
    *   변수(Variable)는 `{{variable_name}}` 형식을 사용하십시오.

---

## 3. Input Formatting (포맷 가이드)

*   **Language**: 원칙적으로 **한국어(Korean)**를 사용하되, 프롬프트 본문이 영어인 경우 그대로 유지합니다.
*   **Tone**: 설명은 명확하고 건조한 기술 문서 톤(Technical Tone)을 유지하십시오.
*   **Description**: 프롬프트의 목적, 사용된 기법(예: Chain-of-Thought, Few-Shot), 예상 입/출력 예시를 포함하면 좋습니다.

---

## 4. Emergency Fallback

*   **스키마 불일치**: 만약 위 컬럼(Property)이 실제 DB에 없다면, 강제로 생성하지 말고 **본문(Body)** 최상단에 `metadata`라는 이름의 Toggle Block을 만들고 그 안에 YAML 형식으로 기입하십시오.
